Introduction

There are:

  • static libraries (*.a)
  • shared libraries (*.so)

A static library is essentialy a set of object files.

How to create an object file?

#include <iostream>

using namespace std;

int foo()
{
  cout << "I'm foo()" << endl;
}
g++ -c tools.cpp

Result: tools.o

How to create a static library?

ar -cq libtools.a tools.o

And check the results (list the content of a specified archive):

$ ar -t libtools.a
tools.o

How to use a library?

#include <iostream>
#include "tools.h"

using namespace std;

int main()
{
  cout << "I'm calling foo()..." << endl;
  foo();
  cout << "I'm back!" << endl;
}
g++ -L . main.cpp -ltools

It's important to type -ltools option after main.cpp. Here's why.

It's worth noting that a naming convention is used here. libtools.a

How to find all static libraries in your system?

find / -name *.a 2>/dev/null
